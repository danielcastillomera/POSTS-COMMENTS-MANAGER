# Internacionalizacion (i18n)

## Idiomas soportados

| Codigo | Idioma |
|--------|--------|
| `es-MX` | Espanol (Mexico) — predeterminado |
| `en-US` | Ingles (Estados Unidos) |

## Implementacion

La aplicacion usa un servicio `I18nService` personalizado construido sobre Angular Signals, sin depender del compilador de i18n de Angular. Esto evita la necesidad de compilaciones separadas por idioma y permite cambiar el idioma de forma instantanea en tiempo de ejecucion.

### Archivos de traduccion

Archivos JSON ubicados en `frontend/src/assets/i18n/`:

- `es-MX.json` - Espanol (Mexico)
- `en-US.json` - Ingles (Estados Unidos)

Usan una estructura de claves anidadas:

```json
{
  "nav": { "posts": "Publicaciones" },
  "posts": {
    "list": { "title": "Publicaciones", "search": "Buscar..." }
  }
}
```

### Uso en plantillas HTML

```html
{{ 'posts.list.title' | translate }}
```

### Uso en TypeScript

```typescript
private readonly i18n = inject(I18nService);

const etiqueta = this.i18n.translate('common.loading');
```

### Selector de idioma

El componente `LanguageSwitcherComponent` en la barra de navegacion permite cambiar entre idiomas. La seleccion persiste entre sesiones mediante `localStorage` con la clave `app_lang`.

### Agregar un nuevo idioma

1. Crear `frontend/src/assets/i18n/{codigo-idioma}.json` con todas las claves requeridas.
2. Agregar la nueva opcion al arreglo `languages` en `LanguageSwitcherComponent`.
3. Actualizar el tipo `Language` en `I18nService`.
