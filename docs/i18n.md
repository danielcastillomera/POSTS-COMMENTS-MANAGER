# Internationalization (i18n)

## Supported Languages

| Code | Language |
|------|----------|
| `es-MX` | Spanish (Mexico) - default |
| `en-US` | English (United States) |

## Implementation

The application uses a custom `I18nService` built on Angular Signals, without Angular's built-in i18n compiler. This avoids the need for separate per-locale builds and makes language switching instantaneous at runtime.

### Translation Files

JSON files located at `frontend/src/assets/i18n/`:

- `es-MX.json` - Spanish (Mexico)
- `en-US.json` - English (United States)

Files use a nested key structure:

```json
{
  "nav": { "posts": "Publicaciones" },
  "posts": {
    "list": { "title": "Publicaciones", "search": "Buscar..." }
  }
}
```

### Usage in Templates

```html
{{ 'posts.list.title' | translate }}
```

### Usage in TypeScript

```typescript
private readonly i18n = inject(I18nService);

const label = this.i18n.translate('common.loading');
```

### Language Switcher

The `LanguageSwitcherComponent` in the top navigation bar allows switching between languages. The selection persists across sessions via `localStorage` under the key `app_lang`.

### Adding a New Language

1. Create `frontend/src/assets/i18n/{lang-code}.json` with all required keys.
2. Add the new language option to `LanguageSwitcherComponent.languages`.
3. Update the `Language` type in `I18nService`.
